NODE.JS&HEXO를 이용한 git blog 만들기

시작에 앞서…

다른 많은 분들이 작성한 글 들이 많지만, 그것을 보고 따라하면서 이해하기엔 이해가 잘 안된 부분들이 있어 직접 과정을 정리하며 내용을 새로 작성했습니다. 혹시라도 처음 본 사람들이 보고 따라합니다면 도움이 될 것이라 생각되지만 추후 버전 업데이트가 되면 이 과정과 불일치 할 수 있음을 미리 알립니다.

이 글을 보면서 따라하면 생성과 배포를 수행 할 수 있지만, 디테일한 부분에서 궁금한 점은 스스로 느끼고 찾아가면서 깨달아 가기를 바랍니다.

또한, tistory에서 작성 한 내용을 옮긴 것으로 일부 깨짐이나 틀어짐이 발생 할 수 있음을 알립니다.

0. NODE.JS와 HEXO

node.js는 자바스크립트를 서버환경에서도 사용 할 수 있도록 해주는 일종의 프로그램이라고 볼 수 있습니다.

말한마디 잘못했습니다간 물어뜯는 사람들도 부지기수이기 때문에 보통 다른 사람들 글을 올린걸 보면 아주 교과서적인 설명글을 올리는게 자주 보이는데 (기술의 영역이니까 당연히 정확한 정보 전달은 중요합니다 ^^) 초보자, 입문자 입장에서 아주 간단하고 쉽게 이해하고자 하면 브라우저에 종속되는 자바스크립트를 서버환경에서도 쓰게 해 줄 수 있는 것이라고 보면 당장 첫 이해의 출발은 편합다. 자세한 것은 관심이 커진다면, 스스로 찾아가면서 개념부터 잡아가는 것을 추천합니다.

hexo는 git blog를 보다 쉽고 편하게 꾸미도록 해주는 플러그인입니다.

이러한 플러그인의 도움을 받지 않는다면, fm으론 사용자가 직접 리포지터리를 따고 클론을 받은 후 하나하나 꾸며주는 과정을 거쳐야합니다. 그러면 웹 퍼블리셔나 프론트엔드 개발자여도 이곳저곳 파일을 만들며 꾸미고 디자인하고 맞추느랴 시간이 많이 걸릴 것입니다. hexo는 이 과정을 줄여주는 역할을 합니다!

1. NODE.JS를 사용자 OS에 맞는 환경에 다운받고, 설치합니다.

https://nodejs.org/ko/download

설치가 정상적으로 되었다면, [node -v] 명령어를 통해 다음과 같이 설치 버전을 확인 할 수 있습니다.

2. hexo를 다운받고, 설치합니다.

  • npm install hexo-cli -g
  • hexo init blog
  • cd blog
  • npm install
  • hexo server

위의 명령어를 순서대로 입력하면 설치 및 hexo 서버가 실행되는데, 주의 할 점은 설치되는 디렉토리는 현재 터미널이 가르키고 있는 디렉토리이므로 잘 생각해서 설치 해야합니다. 본인은 편한 관리를 위해 C:/에서 설치를 진행했습니다.



node.js가 정상 설치 되었다면, npm 명령어를 사용 가능하며 다음과 같이 hexo를 다운받을 수 있습니다.


블로그 초기화 작업을 수행하면, 현재 바라보는 디렉토리에 blog 디렉토리가 새로 생성됩니다..



이 디렉토리에서 생성 된 파일들이 나의 github 리포지터리에 배포되며, 배포전에 로컬 환경에서 미리 볼 수 있습니다.


hexo 서버를 실행 시킨 후, 위의 로컬 주소를 접속하게되면


이처럼 기본 테마를 가진 정적 웹 블로그를 만날 수 있습니다!

3. 테마 적용하기

hexo는 다양한 테마들을 제공하며, 직접 만들어 테마를 배포 할 수도 있습니다.

테마를 새로 만들기엔 디자인도 잘 알아야하고 시간이 꽤 들 것 같아 나는 기존에 공유되어있는 테마들 중, 마음에 드는걸 하나 골라 쓰기로 했습니다.


https://hexo.io/themes/

위 링크에 접속을 하면, hexo에서 지원하는 다양한 테마들을 볼 수 있습니다.

그중에서 마음에 드는 것을 선택하고 설치하고 적용하면 되는데, 나는 [Fluid] 테마가 마음에 들어 이것을 선택했습니다.


테마를 선택하여 들어가면, 그 테마의 github readme.md 를 볼 수 있습니다 !

내가 선택한 테마는 중국에서 만든 테마 같은데, 다행이 영어를 지원해서 설치하는데 큰 어려움이 없었습니다.

스크롤을 내려보면 다음과 같이 간편한 설치방법을 안내해주며, 잘 따라하면 테마의 기본팩은 설치가 끝납다.

주의할 점은, 마찬가지로 설치되는 경로는 ‘현재 터미널에서 가르키는 경로’ 이므로 hexo blog의 테마니까 hexo blog로 이동 후 설치를 진행해야 합니다. 이 과정을 따라하고 있다면, C:\blog로 이동 후 설치를 하면 되며 별도 디렉토리에 설치하고 있다면, 설치하고자 하는 blog 디렉토리에서 설치하면 됩니다.. 별도 디렉토리에 설치할 사람이라면 이걸 안보고 알아서 잘 설치 할듯


위와같이 blog\node_modules에 [hexo-theme-fluid] 디렉토리가 새로 생기며, 설치가 됩니다!
설치 문서를 읽어보면 Way A, B가 존재하는데 둘중에 편한 방법을 통해 fluid 설정파일을 만들어주면 됩니다..

A는 fluid 테마 전용 yml 파일을 만든 후 링크의 내용을 복사하여 저장하는 것이며, B는 fluid 테마 전용 yml파일을 다운받아 저장하는 것입니다. 나는 A방법을 따르기로 했습니다.

파일을 생성하고나면, _config.yml에서 어떤 테마로 generate 할 것인지 내용을 수정해줘야 합니다.


theme : 부분이 기본값으로 landscape로 되어있을텐데, 이를 fluid로 바꿔줍니다.

그 후, 꼭 [hexo new page about] 를 통해 about 페이지를 만들어 줘야합니다 !

이 과정을 놓친다면 fluid 테마가 제대로 적용되지 않습니다.


이렇게 나온다면 테마가 제대로 적용되지 않았습니다.


제대로 적용되었다면 다음과 같이 fluid 테마의 대한 내용이 나옵니다. ^^


과정을 잘 따라했다면, 다음과 같이 hexo fluid 테마의 기본화면을 볼 수 있습니다!

4. 이미지 교체 및 메뉴 한글화

기본 설정으로 글 제목과 내용만 쭉 작성 후, 배포를 할 것이라면 이 과정을 생략해도 됩니다..

하지만 나는 메뉴를 한글로 바꾸고 싶고 이미지도 바꾸고 싶었는데, 같은 생각인 사람은 이 과정을 따라하면 도움이 됩니다..

우선 이미지를 바꾸려면 테마가 설치된 디렉토리를 찾아가서 소스의 img를 찾아 바꿔주어야 합니다.

만약 기본으로 제공하는 이미지는 그냥 두고 내가 원하는 새로운 이미지만 추가하여 바꿀 것이라면 파일을 이곳에 넣어준 뒤 _config.fluid.yml 파일에서 파일명을 수정해주면 됩니다.. 나의 경우엔 기본 이미지는 두고, top_banner.jpg 라는 이름으로 새로 추가했습니다. 참고로 이미지는 16:9의 fhd 비율이 보기좋게 나옵니다. ^^


_config.fluid.yml에서 다음과 같이 모든 영역에 대해 찾고, 수정해야 합니다. 서브라임 텍스트를 비롯한 전문 에디터에선 모든 부분을 찾아 일괄 수정하는 것이 가능하므로, 메모장 수작업보다 이런식으로 작업하는 것을 추천합니다!
언어 수정은 사용자가 직접 해당 언어 파일을 만들어 줘야하는데, 보통 테마 내부에 [languages] 디렉토리를 가면 i18n 기반의 언어 파일들이 있습니다. 이 중 하나를 복사하여 새로 만들면 됩니다..



en 파일을 복붙하여 ko파일로 새로 만들어 준 후, 파일을 열어 해당하는 부분을 수정하면 됩니다..



나는 메뉴명을 한글화 하는 정도만 했습니다. 간단한 조작으로 메뉴 추가도 할 수 있을 것 같은데, 그것에 대해선 좀 더 알아가야 할 듯 합니다.


바뀐 이미지 확인과 메뉴명이 바뀐 것을 확인 할 수 있습니다.

이 외, 메뉴마다 접속시 뜨는 메세지나 좌측상단의 title 같은 것들은 f12 개발자 모드를 통해 노드를 선택하거나 해당되는 텍스트 문구를 _config.fluid.yml 파일에서 찾아 수정해주면 됩니다..

generate에서 에러가 발생하지 않았다면, 잘 적용되는 것을 볼 수 있습니다.

5. GITHUB에 블로그 배포하기

이제 모든 세팅이 다 되었다면, 깃허브에 블로그를 배포하는 작업을 하도록 합니다.

제일 먼저, 깃허브에서 github.io 의 이름을 가진 리포지터리를 새로 만들어야 합니다.


작성자의 경우엔 이미 생성되어있어서 만들 수 없지만, 처음 시도하는거라면 보통은 만들어질 것입니다 !

이후 config.yml 파일에서 깃 블로그에 보여질 title같은 기본 정보를 바꿔주고, deploy에 필요한 정보도 바꿔줍니다.

설정이 끝났다면, 이제 deploy만 해주면 됩니다..

정상 배포되었다면 위와 같이 github.io 생성한 링크로 접속이 잘 되고, 리포지터리에 갱신 된 것을 볼 수 있습니다 ^^

블로그 글 작성은 hexo new post {title} 명령어를 통해 새로운 글 파일을 만들어 주고, 그 안의 title 및 내용을 바꾸어주면 됩니다..

제 글만으로 이해가 잘 되지 않는다면, 제가 작성하면서 참고한 아래의 링크를 읽으시면서 이해하는 과정을 거치시는 것을 추천드립니다 !
